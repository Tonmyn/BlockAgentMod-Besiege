BlockAgentMod-Besiege (BAM)
为 Besiege 注入大语言模型（LLM）能力的方块智能体框架。



📖 简介
BlockAgentMod (简称 BAM) 是一个基于 Besiege 开发的 AI 交互 Mod。它的初衷是在 Besiege 的沙盒环境中建立一个与外部大语言模型通信的桥梁。

目前，BAM 以一个游戏内 AI 终端的形式存在，允许玩家直接在游戏界面中与自定义的 AI 模型进行对话。我们称之为 “Agent” 而不是 “Copilot”，是因为未来的规划中，它将不仅限于对话，而是能够感知游戏状态并采取行动。

✨ 当前功能 (v1.x)
🎮 游戏内集成 UI：无需切出游戏，通过快捷键呼出聊天窗口。
🔌 兼容 OpenAI API 格式：支持任何兼容 OpenAI 接口协议的模型（如 ChatGPT, Claude, 本地部署的 LLM 等）。
🔑 本地配置存储：API 地址和 Key 在本地安全保存，重启游戏无需重复输入。
🛠️ 底层兼容性处理：针对 Unity 5.4.5 的底层安全限制进行了特殊适配，确保网络请求稳定发出。
🚀 安装指南
确保你的游戏已经安装了必要的 Mod 加载器（如 BepInEx 或其他适用于你游戏版本的框架）。
从 Releases 页面下载最新版本的 BlockAgentMod.dll。
将 DLL 文件放入游戏的 Mods (或对应的插件 Plugins) 文件夹中。
启动游戏。
🕹️ 使用方法
进入游戏后，按下呼出 UI 的快捷键（默认：F5，可在配置文件中修改）。
在设置面板中填入你的：
API URL (例如：https://api.openai.com/v1/chat/completions)
API Key (例如：sk-xxxx...)
模型名称 (例如：gpt-3.5-turbo)
在输入框中输入你的问题，按下发送，等待 AI 回复。
🗺️ 未来愿景
目前 BAM 只是实现了“感知”的第一步（文本输入输出）。未来的版本计划引入本地 Python 中间件服务，以实现真正的智能体闭环：

状态感知：通过中间件读取 Besiege 当前的机器结构、方块类型。
逻辑分析：AI 根据当前机器状态进行受力分析、平衡性检测或提供改造建议。
行动控制：最终演变为真正的 Copilot，辅助甚至自动调整机器参数。
⚠️ 注意事项
网络延迟：AI 的响应时间取决于你使用的 API 服务器和网络状况，请求期间游戏不会卡顿。
API 费用：本 Mod 仅作为通信管道，调用第三方 API 可能会产生网络流量或 token 费用，请自行承担。
隐私安全：你的 API Key 仅存储在你本地的电脑上，且网络请求直接由你的电脑发往 API 服务器，不经过任何第三方中间人。
🛠️ 开发者说明
本 Mod 针对 Besiege 特定的 Unity 5.4.5 环境开发。由于旧版引擎的安全沙箱限制（如禁用原生 UnityEngine.WWW 类），网络通信模块采用了特殊的底层调用策略以绕过限制并保证内存安全。
